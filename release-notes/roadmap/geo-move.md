# Move your organization to a different geography

We will be gradually rolling out again the ability to move an existing Azure DevOps organization to a different geography (geo move). Only moves between geographies will be supported. This would help customers meet data residency requirements by keeping their organization's data in the geography they need. We'll share more details as well, as the work progresses.
